
1) CaseChanging - класс для смены падежей слова с именительного на какой-то другой. ( На данный момент не используется, так как требуется обратное, т.е. при подстановке слов в разных падежах получать форму в именительном падеже. )

2) Crawler - класс необходим для парсинга сайтов. 
  a) Метод Search принимает в качестве параметра ArrayList с url, которые надо посетить. 
  b) Переменная pagesToVisit содержит список сайтов, которые еще может програма посетить.
  с) Переменная pagesVisited содержит в себе страницы, которые программа уже посетила ( так как это Set, то дубли отсутствуют)
  d) Метод nextUrl выполняет сравнение существуют ли еще страницы, которые нужно посетить.

3) CrawlerLeg - класс, который осуществляет подключение к сайтам и ищет на них все ссылки внутри домена, куда парсер подключается. (Например, при подключении к maps.google.ru в переменную links будут добавляться сайты maps.google.ru/map1 или maps.google.ru/blablalb и т.д, но не google.ru )

4) Filereader - класс для чтения списка сайтов или списка ингредиентов, записанных в текстовом документе. Необходим лишь тогда, если шаблоны сайтов, которые нужно пропарсить будут идентичны. ( На данный момент класс не используется)

5) GetIngredientsFromTheSite - класс берет список ингредиенты с сайта http://www.bazareceptov.ru/ в именительном падеже и заносит их в переменную fullListOfIngredients типа List.

6) HttpsClient - класс создан для коннекта к сайтам по https. Временно не используется
 
7) MongoConnector - класс необходим для подключения к базе данных и передаче рецептов или ингредиентов полученных с разных сайтов.

8) ParsingSay7Info - класс для парсинга 1 сайта, так как шаблон у многих сайтов уникален, следовательно логику необходимо прописывать для каждого сайта. 

9) WebDriverExample - класс для тестирования WebDriver, чтобы работать с динамическими данными и js. (временно не используется)
